Now that we have created a web site using a single web server, we will extend the solution to include a second web server and a load balancer.
When web surfers attempt to access our web site, the load balancer will spread the requests across our web instances.

Create a Load Balancer

  1. To add a load balancer, select the Load Balancers link in the navigation pane.

Add a Load Balancer Pool

  1. The main view lists the Load Balancer Pools for this project. At present, the list should be empty.

    NOTE: The Pool is a container that holds instances

  2. To create a new Load Balancer Pool, select + Add Pool. A pop-up window will appear.

Add a Load Balancer Pool. Step 1

  1. In the Name field, type a name for your pool. A good choice is web-pool
  2. In the Subnet drop-down menu, select the private network.

    The private network should be on the only choice in the drop-down list.

  3. In the Protocol drop-down menu, select HTTP
  4. In the Load Balancing Method drop-down menu, select ROUND_ROBIN

    All other fields can be left to their default values.

  5. Once completed, select Add to add your Load Balancer Pool.

    You will be returned to the Load Balancer view.

Load Balancer Pool Created.

  1. Once completed, a new pool entry will appear in the Pools list.

    The name should be web-pool

  2. The next step is to add a VIP to the pool. To do this, select More then Add VIP.

    A pop-up window will appear.

  3. NOTE: The VIP is the IP address that users will type into their web browser to reach your web site.

Add a VIP. Step One

  1. In the Name field, type a name for your VIP. A good choice is web-vip.
  2. In the Specify a free IP address ... field, type a free IP address. A good choice is 192.168.55.35.
  3. In the Protocol Port field, type 80
  4. In the Protocol drop-down menu, select HTTP
  5. In the Session Persistence drop-down menu, select SOURCE_IP
  6. Once completed, select Add to add your VIP.

    You will be returned to the Load Balancer view.

VIP Added

  1. Once completed, the VIP name (web-vip) will appear in the web-pool Pools entry.
  2. The next step is to add instance members to the pool. To do this, select the Members tab.

Add First Pool Member

  1. The main view lists the instance members in the Load Balancer Pool.

    At present, the list should be empty.

  2. To add a new instance Member to the Load Balancer pool, select + Add Member.

    A pop-up window will appear.

Add First Pool Member. Step One

  1. In the Pool drop-down menu, select web-pool.
  2. In the Members selection, tick web-app-instance.
  3. In the Protocol Port field, type 80.
  4. Once completed, select Add to add the member instance.

    You will be returned to the Members view.

First Pool Member Added.

  1. Once completed, the new member instance will be added to the web-pool Pool.
  2. The next step is to add a public IP address to the Pool. To do this, select the Access & Security link in the Navigation pane.

Add Public IP Address. Step One

  1. If not already selected, select the Floating IPs tab to display the list of available floating IPs.

    At this stage, there will be only one Floating IP allocated. We will need to allocate another IP for our Load Balancer.

  2. To allocate another Floating IP, select the Allocate IP To Project button.

    A pop-up window will appear.

Add a Public IP Address. Step Two

  1. The Project Quotas should indicate that one Floating IP has been assigned (blue bar) and that you are requesting a second Floating IP (green bar).
  2. Select the Allocate IP to allocate the IP address to the project.

    You will be returned to the Floating IPs view.

Public IP Address Added

  1. On completion, a second Public IP address will be allocated to the project.
  2. To associate the Public IP address with the VIP, select Associate.

    A pop-up window will appear.

Associate Public IP Address

  1. In the Port to be associated drop-down menu, select the None: 192.168.55.35.

    This will associate the Public IP address to the VIP.

  2. To complete the association, select the Associate button.

    You will be returned to the Floating IPs view.

Public IP Address Associated

  1. On completion, the IP address will be associated with the VIP.

    NOTE: In the Instance column, the value should be “-“.

  2. The next step is to add a second web server instance. To start, select Instances from the Navigation pane.

Add Second Instance to Pool

  1. On completion, a second web server instance will appear in the Instances list.
  2. The next step is to add this second web server instance to the load balancer. Select Load Balancers link in the Navigation pane.

Add Second Instance to Pool. Step One

  1. If it is not already selected, select the Members tab.
  2. The existing web server instance should be in the Member list.
  3. To add the second web server instance, select + Add Member.

    A pop-up window will appear.

Second Instance Added to Pool

  1. On completion, there should be two instance members in the Members list.
  2. The next step is to check out the website!

    If you need to recall the IP address, select Access & Security.

Recall Public IP Address

  1. If not already selected, select the “Floating IPs” tab.
  2. Copy the IP address of the VIP, open a new tab in your web browser and paste in the IP address
  3. Try to access the IP Address from different web browsers to see if you can get different responses.
